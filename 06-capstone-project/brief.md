# Capstone Brief — The Pre-CSS Checkpoint

This is the final gate before moving to CSS. Build one complete, multi-section page from scratch that uses everything covered so far. Treat this like a small real project, not a worksheet.

## The scenario

Pick one: a small business's one-page website, a community event page, or a personal portfolio. (If none of these fit, propose your own — just get it approved first.)

## Requirements — the page must include

- [ ] Correct document structure: `<!DOCTYPE html>`, `<html lang="en">`, `<meta charset>`, `<meta viewport>`, a real `<title>`
- [ ] One `<h1>`, with all other headings in correct unskipped order
- [ ] `<header>` with a `<nav>` containing at least 3 working anchor links
- [ ] `<main>` containing at least two `<section>` or `<article>` regions
- [ ] At least one image with genuinely descriptive `alt` text
- [ ] At least one list (`<ul>` or `<ol>` — your choice, but it must be the *right* choice for the content)
- [ ] One `<table>` used for real tabular data (not layout)
- [ ] One `<form>` with at least 3 different input types, every one with a correctly matched `<label>`
- [ ] `<footer>` with contact info, including a real `mailto:` link
- [ ] Zero `<div>` tags used where a semantic tag would fit better
- [ ] Passes the W3C Validator with zero errors

## Submission

1. The `.html` file itself.
2. A short paragraph (3–5 sentences) explaining one tag choice you had to actually think about — not just "I used `<nav>` for navigation," but a case where two options seemed plausible and you had to decide.

## What happens after this

Once a page clears this checklist, that student is ready to start layering CSS onto their own work — meaning their *own* pages from this pack become the canvas for the styling unit, not a fresh generic template. See `rubric.md` for how this gets scored.
