# Capstone Rubric

Score each category 0–2. 8+ out of 10 = ready for CSS. Below that, revisit the specific weak category using the matching folder in this pack before moving on.

| Category | 0 | 1 | 2 |
|---|---|---|---|
| **Document structure** | Missing doctype/lang/meta tags | Present but with errors (wrong order, typos) | Fully correct |
| **Heading hierarchy** | Skipped levels or multiple `<h1>`s | Mostly correct, one slip | Perfect single `<h1>`, correct order throughout |
| **Semantic layout** | Div-soup, no landmarks | Some semantic tags, some divs used where they shouldn't be | Fully semantic, zero unnecessary divs |
| **Images & lists & tables** | Missing alt text, wrong list type, or table misused | One of the three is off | All three used correctly and purposefully |
| **Forms & accessibility** | Labels missing or unmatched, wrong input types | Mostly correct, 1–2 mismatches | Every label matched, correct types, keyboard-navigable |
| **Validator result** | Multiple errors remain | 1–2 minor errors | Zero errors |

## Using this diagnostically, not just as a grade

If most of the class scores low in the same category, that's more useful information than any individual grade — it tells you exactly which concept needs a second pass before CSS, and you can point back to the matching exercise:

- Weak on structure/headings → revisit `01-debugging-exercises`
- Weak on independent tag choice → revisit `02-blueprint-to-html`
- Weak on conceptual "why" → revisit `03-diagnostic-quiz`
- Weak on catching their own mistakes → lean harder on `04-peer-review-checklist`
